# Coding Standards

## Style Preferences

- **Indentation**: Use 2 spaces (not tabs).
- **Async**: Prefer `async/await` over promises.
- **Imports**: Avoid using barrel imports.
- **Modifications**: Prefer modifying existing files over creating new ones.

## Naming Conventions

- **Variables/Functions**: Use `camelCase`.
- **Classes/Components**: Use `PascalCase`.
- **Constants**: Use `SCREAMING_SNAKE_CASE`.
- **Variables**: Always use `const` or `let`, never `var`.
